Manual para desarrollar una página Web accesible para adultos mayores

A la hora de interactuar con páginas Web, a los adultos mayores se les pueden presentar ciertas dificultades y barreras que les impiden aprovechar al máximo lo que estas ofrecen. Refiérace a la sección accesibilidad para adultos mayores de la página de Conceptos para entender más sobre estas dificultades.

El objetivo de esta página es servir de manual o guía para la creación y desarrollo de páginas Web que son accesibles para adultos mayores en su totalidad. Para ello, se utilizan varios recursos presentes en material adicional. El principal es el WCAG 2.1, que son recomendaciones y lineamientos a cumplir para que una página Web sea accesible. Para alcanzar dichas recomendaciones se utilizan además dos herramientas: Accessible Rich Internet Applications (ARIA), que son un conjunto de atributos de HTML que ofrecen información útil a lectores de pantalla y otras tecnologías sobre el contenido de una página que de lo contrario no sería posible de identificar, y también la herramienta WAVE, que sirve para evaluar el nivel de accesibilidad de una página Web existente. Más adelante se entrará en detalle con estas herramientas.

1. Desarrollo Web accesible

Al desarrollar una página Web, lo más importante es tener en mente implementar la accesibilidad desde antes de comenzar a crearla y durante todo su desarrollo. Así es más fácil considerar en todo momento los aspectos que hacen una Web accesible, sin tener que volver a escribir código cuando ya es demasiado tarde. Por cada componente, texto, información o demás que sea agregado a la página se debe pensar: ¿es comprensible? ¿su función es clara? ¿cualquier persona puede identificarlo, entenderlo y/o usarlo? ¿se puede interactuar con cualquier medio (teclado, mouse, pantalla táctil, voz, etc)? ¿su contexto y posición respecto al resto de la página es adecuado? ¿es necesario?

Web Content Accessibility Guidelines 2.1 o WCAG 2.1 son las actuales recomendaciones dadas por el World Wide Web Consortium (W3C) para que una página Web sea accesible. Se pueden encontrar 4 principios que toda página web debe tener para considerarse accesible, que son:

  • Perceptible: Los componentes de la interfaz de usuario deben ser mostrados a los usuarios en formas que ellos puedan entender.
  • Operable: Los componentes de la interfaz de usuario y la navegación debe ser manejable.
  • Comprensible: La información y las operaciones de los usuarios deben ser comprensibles.
  • Robusto: El contenido debe ser suficientemente robusto para que pueda ser interpretado por una variedad de agentes y tecnologías asistivas.

A continuación se mostrarán los puntos más importantes incluidos en estos 4 principios, sus técnicas y recomendaciones así como también errores que no se deben cometer.

1.1. Perceptible

Todos los componentes de una página Web deben poder ser identificados y entendidos por todos los usuarios. Muchos adultos tienen dificultades con la visión, y letras demasiado pequeñas, colores con bajo contraste y demás factores dificultan su percepción de las páginas. Algunos utilizan lectores de pantalla para percibir la información, por lo que no es posible concentrarse únicamente en el diseño visual de la página. Además, otros tienen dificultades auditivas, por lo que demostrar información únicamente por audio o videos sin subtítulos no es viable.

1.1.1. Alternativas textuales

Todo contenido de una página que no se basa en texto debe tener alternativas textuales que pueden ser traducidos por el medio de elección del usuario. El ejemplo más claro de esto es en el uso de imágenes. Toda imagen incluida en una página debe ofrecer un texto alternativo para que usuarios que no puedan verla puedan comprender su contenido. Con HTML, el elemento img, usado para introducir imagenes a una página, contiene el atributo alt. El texto ingresado a dicho atributo servirá de texto alternativo para la imagen.
